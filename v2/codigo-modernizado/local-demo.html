CardDemo - Aplicação Modernizada

Demonstração da arquitetura modernizada do sistema CardDemo

Sobre o Projeto

Esta é uma demonstração da modernização da aplicação AWS CardDemo, migrando de um sistema mainframe Cobol/CICS/VSAM para uma arquitetura moderna baseada em microserviços Java Spring Boot, React e PostgreSQL.

A demonstração completa seria implementada com containers Docker, mas esta versão estática mostra os componentes e suas interações.

Arquitetura de Microserviços

A aplicação modernizada utiliza uma arquitetura de microserviços com os seguintes componentes:

Frontend (React)

Interface de usuário moderna e responsiva para interação com o sistema.

API Gateway (Nginx)

Roteamento de requisições para os microserviços apropriados.

Auth Service

Autenticação e autorização de usuários com JWT.

Customer Service

Gerenciamento de dados de clientes.

Account Service

Gerenciamento de contas de cartão de crédito.

Card Service

Gerenciamento de cartões de crédito.

Transaction Service

Processamento e consulta de transações.

PostgreSQL

Banco de dados relacional para persistência.

Configuração Docker Compose

Em um ambiente completo, a aplicação seria executada com a seguinte configuração:

services:
  # Frontend (React)
  frontend:
    image: carddemo-frontend
    ports:
      - "80:80"
      
  # API Gateway
  nginx:
    image: nginx
    ports:
      - "8080:80"
      
  # Database
  postgres:
    image: postgres
    environment:
      POSTGRES_DB: carddemo
      
  # Microservices
  auth-service:
    image: carddemo-auth-service
    depends_on:
      - postgres
      
  customer-service:
    image: carddemo-customer-service
    depends_on:
      - postgres
      
  account-service:
    image: carddemo-account-service
    depends_on:
      - postgres
      
  card-service:
    image: carddemo-card-service
    depends_on:
      - postgres
      
  transaction-service:
    image: carddemo-transaction-service
    depends_on:
      - postgres

Funcionalidades do Sistema

O sistema modernizado oferece as seguintes funcionalidades principais:

  • Autenticação e Segurança: Login, autorização baseada em papéis, tokens JWT
  • Gestão de Clientes: Cadastro, consulta e atualização de clientes
  • Gestão de Contas: Criação e manutenção de contas de cartão de crédito
  • Gestão de Cartões: Emissão, ativação e bloqueio de cartões
  • Processamento de Transações: Registro e consulta de transações
  • Relatórios: Geração de extratos e relatórios analíticos

Implementação Real

Para implementar completamente esta aplicação, seriam necessários os seguintes passos:

  1. Configurar o ambiente Docker para construção e execução de containers
  2. Desenvolver os microserviços Java Spring Boot com suas APIs
  3. Criar o frontend React para interação com os usuários
  4. Configurar o banco de dados PostgreSQL e seus schemas
  5. Implementar a segurança com Spring Security e JWT
  6. Migrar os dados do sistema legado Cobol para o novo sistema
  7. Realizar testes de integração e validação
